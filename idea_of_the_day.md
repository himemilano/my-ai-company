# アプリ仕様書: 今日の小さな勲章

## 1. はじめに

本ドキュメントは、PMより提案された「今日の小さな勲章」アプリの機能と要件を定義するものです。現代人の日常に「小さな達成感」と「ささやかな変化」をもたらし、自己肯定感を高めることを目的とします。ブラウザだけで手軽に利用できるWebアプリケーションとして開発します。

### 1.1. アプリ概要

*   **アプリ名**: 今日の小さな勲章（きょうのちいさなくんしょう）
*   **ターゲット層**:
    *   日常に漠然とした虚無感を感じる人
    *   何か新しいことを始めたいが、一歩が踏み出せない人
    *   日常のマンネリを打破したいが、大それた行動はしたくない人
*   **解決する課題**:
    *   「今日も何もしなかった」という自己肯定感の低下
    *   新しい行動への一歩を踏み出せないジレンマ
    *   日常のマンネリ化と選択疲れ

### 1.2. 動作環境

*   主要なモダンブラウザ (Chrome, Firefox, Safari, Edge)

## 2. 機能一覧

本アプリは以下の主要な3つの機能で構成されます。

1.  **「今日のミッション」生成**: 超簡単なミッションをランダムに表示します。
2.  **達成報告＆勲章獲得**: ミッション達成を祝い、履歴に記録します。
3.  **「明日のヒント」表示**: 達成後に明日に繋がるゆるいヒントを提示します。

## 3. 機能詳細

### 3.1. 「今日のミッション」生成

#### 3.1.1. 概要

ユーザーがメイン画面のボタンをクリックすると、ランダムに選ばれた一つの超簡単なミッションが表示されます。

#### 3.1.2. ユーザーインターフェース (UI)

*   **メイン画面 - 初期状態**:
    *   アプリ中央に大きく「今日の小さな勲章をGET！」ボタンを配置します。
    *   既にその日のミッションが生成済みの場合（ローカルストレージに当日ミッション情報が存在する場合）、ミッション内容と「達成！」ボタンが表示されます。
*   **ミッション表示状態**:
    *   「今日の小さな勲章をGET！」ボタンクリック後、または再訪問時にミッションが既に生成されている場合、以下の要素が表示されます。
        *   ミッションタイトル：「今日のミッション」
        *   ミッション内容: ランダムで選ばれた具体的なミッションテキスト
        *   「達成！」ボタン

#### 3.1.3. ロジック

1.  **ミッションリスト**: 以下の基準を満たすプリセットされたミッションリストからランダムに1つを選択します。
    *   **基準**: 「誰にでもできる」「お金がかからない」「特別なスキル不要」「数分で完了」
    *   **ミッション例 (抜粋/追加)**:
        *   「今日はいつもより5分だけ早く起きて、空の写真を撮ってみる」
        *   「普段使わないカフェの席に座ってみる」
        *   「冷蔵庫の奥に眠っている調味料を一つ使ってみる」
        *   「帰り道、いつもと違う一本裏の道を歩いてみる」
        *   「誰かにポジティブな言葉を一つかける（心の中で思ってもOK！）」
        *   「スマホのホーム画面を少しだけ整理する（アプリを一つ動かすだけでもOK）」
        *   「普段あまり聞かないジャンルの音楽を聴いてみる」
        *   「目についた広告のコピーを声に出して読んでみる」
        *   「1日1回だけ、空を見上げて深呼吸する」
        *   「今日の夜、窓の外の星を一つ探してみる」
        *   「使っていないペンを一本、別の用途に使ってみる」
2.  **ミッション生成のタイミング**:
    *   「今日の小さな勲章をGET！」ボタン押下時、またはアプリ訪問時にその日のミッションが未生成の場合。
    *   1日1回のみ生成可能とします。既にその日のミッションが生成されている場合は、既存のミッションを表示します。
3.  **データ保存**: 生成されたミッションは、その日の日付と共にローカルストレージに一時保存されます。
    *   `localStorage.setItem('currentMission', JSON.stringify({ date: 'YYYY-MM-DD', text: 'ミッション内容' }))`

### 3.2. 達成報告＆勲章獲得アニメーション

#### 3.2.1. 概要

ユーザーがミッションをクリアし「達成！」ボタンをタップすると、視覚的な達成感を演出するアニメーションが表示され、その日の勲章が履歴として記録されます。

#### 3.2.2. ユーザーインターフェース (UI)

*   **アニメーション**:
    *   「達成！」ボタンタップ後、画面全体に紙吹雪やキラキラしたエフェクトが短時間（例: 2〜3秒）表示されます。
    *   同時に「おめでとう！今日の小さな勲章をGETしました！」というメッセージが中央に表示されます。
*   **勲章表示**:
    *   アニメーション終了後、またはアニメーションと並行して、その日の日付とミッション内容が刻まれた、シンプルな「今日の勲章」アイコンが画面下部に表示されます。
*   **履歴表示エリア**:
    *   画面下部などに、これまでに獲得した勲章の履歴が「直近7日間」表示されるエリアを設けます。
    *   履歴には日付とミッション内容が表示されます。新しいものから順に表示します。

#### 3.2.3. ロジック

1.  **達成判定**: 「達成！」ボタン押下でミッション達成とみなします。
2.  **アニメーション再生**: CSSアニメーションやJavaScriptを用いて紙吹雪、キラキラエフェクト、メッセージ表示を演出します。
3.  **勲章データ保存**:
    *   達成されたミッションは、日付、ミッション内容、達成時刻を伴ってローカルストレージに保存されます。
    *   データ構造例: `localStorage.setItem('achievements', JSON.stringify([{ date: 'YYYY-MM-DD', mission: 'ミッション内容', timestamp: 'ISO_TIME_STRING' }, ...]))`
    *   1日1勲章の原則を維持します。もし同じ日に複数回「達成！」された場合でも、その日の勲章は一つとします（初回達成時の情報を使用）。
4.  **履歴表示**: ローカルストレージに保存された `achievements` データから、直近7日間のデータを抽出し、画面に表示します。表示順は降順（新しいものが上）。

### 3.3. 「明日のヒント」機能

#### 3.3.1. 概要

今日のミッションを達成した後にのみ、ユーザーの視点を少しだけ変えるための「明日に繋がる小さなヒント」がランダムに提示されます。

#### 3.3.2. ユーザーインターフェース (UI)

*   **表示位置**: 勲章獲得アニメーション後、勲章表示の下あたりに、「明日のヒント」というタイトルと共にヒント内容が表示されます。
*   **UI要素**: シンプルなテキスト表示。強調しすぎず、強制力がないことを示すデザインにします。
*   **ボタン**: 特にアクションを促すボタンは不要です。

#### 3.3.3. ロジック

1.  **表示条件**: 今日のミッションが「達成済み」である場合にのみ表示します。
2.  **ヒントリスト**: 以下の基準を満たすプリセットされたヒントリストからランダムに1つを選択します。
    *   **基準**: 「日々の視点を少しだけ変えるための提案」であり、強制力を持たないゆるい内容。
    *   **ヒント例 (抜粋/追加)**:
        *   「明日は、街で見かける『青いもの』に少しだけ注目してみよう」
        *   「明日は、一つだけ子どもの頃の思い出を振り返ってみよう」
        *   「明日は、いつもより少しだけゆっくり息を吸ってみよう」
        *   「明日は、普段通らない道を少しだけ意識して歩いてみよう」
        *   「明日は、雲の形をゆっくり観察してみよう」
        *   「明日は、自分の好きなところを一つ見つけてみよう」
        *   「明日は、いつもより少しだけ感謝の気持ちを意識してみよう」
        *   「明日は、目覚ましが鳴る前に一度、伸びをしてみよう」
3.  **データ保存**: ヒントはローカルストレージに保存せず、表示時のみ生成・提示で問題ありません。

## 4. 非機能要件

### 4.1. パフォーマンス

*   ブラウザ上での動作が軽快であること。アニメーションもスムーズに再生されること。
*   ローカルストレージへの読み書きは高速に行われること。

### 4.2. アクセシビリティ

*   基本的なキーボード操作に対応すること（ボタンのTabキー移動、Enterキーでのクリックなど）。
*   テキストコントラストを適切に確保すること。

### 4.3. 堅牢性

*   ローカルストレージのデータが破損した場合でも、アプリがクラッシュしないようにする（データリセットや初期化処理の考慮）。
*   日付変更時のミッションリセット処理が適切に動作すること。

## 5. 技術スタック (想定)

*   **フロントエンド**: HTML5, CSS3, JavaScript (Vanilla JS を推奨。必要に応じて軽量なライブラリ/フレームワークの検討も可)
*   **データ永続化**: Web Storage API (localStorage)

## 6. 今後の拡張性 (任意)

*   ミッション・ヒントのカテゴリ分けとフィルタリング機能
*   過去の勲章をカレンダー形式で表示する機能
*   ユーザーが独自のミッションを追加できる機能
*   ミッションの難易度設定と、それに応じた勲章デザインの変化
*   SNSシェア機能（達成した勲章を画像として共有）
*   通知機能（毎日のミッション生成リマインダー、Web Push APIなど）

## 7. 開発タスクの切り出し (例)

### 7.1. フロントエンド

*   **基本構造とスタイリング**
    *   HTMLの骨格作成 (メインコンテナ、ボタン、ミッション表示エリア、勲章履歴エリアなど)
    *   CSSによる全体のレイアウト、カラースキーム、タイポグラフィの定義
    *   ボタン、ミッション表示、勲章アイコンなどの要素別スタイリング
    *   アニメーション (紙吹雪、キラキラエフェクト、メッセージ表示) 用CSS/JS実装
*   **JavaScriptロジック**
    *   **メイン機能**:
        *   「今日の小さな勲章をGET！」ボタンのイベントリスナー実装
        *   ミッションリストからのランダム選択とDOMへの表示
        *   「達成！」ボタンのイベントリスナー実装
        *   「明日のヒント」リストからのランダム選択とDOMへの表示
    *   **データ管理**:
        *   `localStorage` を用いた `currentMission` (今日のミッション) の保存・読み込み・更新ロジック
        *   `localStorage` を用いた `achievements` (達成履歴) の保存・読み込み・追加ロジック
        *   日付に基づいたミッションの有効期限チェックとリセット処理 (日が変わったら新しいミッションを生成可能にする)
    *   **UI/UX**:
        *   各機能表示のON/OFF切り替えロジック (例: ミッション表示中はGETボタンを非表示にする)
        *   勲章履歴の動的なDOM生成と表示 (直近7日間)

### 7.2. データ設計

*   **ミッションリストの定義**: JavaScriptオブジェクトの配列、またはJSONファイル形式
*   **ヒントリストの定義**: JavaScriptオブジェクトの配列、またはJSONファイル形式
*   **`localStorage` キーとデータ構造の定義**:
    *   `currentMission`: `{ date: 'YYYY-MM-DD', text: 'ミッション内容' }`
    *   `achievements`: `[{ date: 'YYYY-MM-DD', mission: 'ミッション内容', timestamp: 'ISO_TIME_STRING' }, ...]`
