# アプリケーション仕様書: Tab-Zen

## 1. 概要
「Tab-Zen」（タブゼン）は、現代人のデジタル環境における情報過多とそれに伴う集中力散漫、軽度な意思決定疲れを解消する、ブラウザ上で動作するミニマルなWebアプリケーションです。本アプリケーションは、ワンクリックでブラウザの全タブを一時停止し、クリーンな作業空間と一時的な思考整理ツール、集中支援タイマーを提供することで、ユーザーがデジタルノイズから瞬時に離脱し、意識的に「リセット」する習慣を形成することを目的とします。

## 2. 目的
PCを使って作業する人々が、デジタル環境の混沌から一時的に離脱し、意識的に「リセット」することで、集中力を高め、思考を整理できるシンプルかつ効果的なツールを提供すること。これにより、情報過多によるストレスを軽減し、生産性向上を支援します。

## 3. ターゲットユーザー
PCを使って作業するすべての人。特に、複数のプロジェクトやタスクを同時に進めることが多く、ブラウザのタブが常に数十個開いているビジネスパーソン、クリエイター、学生。集中したい時にデジタルな「一時停止」を求め、頭の中を整理するためのクリーンな空間が欲しいと感じている人々。

## 4. 主な機能

### 4.1. 機能1: ワンクリック「全タブ一時停止」機能

#### 4.1.1. 機能概要
ブラウザの拡張機能アイコンをワンクリックするだけで、現在開いている全てのタブを瞬時に非表示にし、専用のミニマルな画面に切り替えます。これにより、視覚的な情報過多から一瞬で解放され、デジタルデトックス状態を促します。保存されたタブは、ミニマル画面からいつでも元の状態にワンクリックで復元可能です。

#### 4.1.2. 詳細仕様
*   **トリガー:** ブラウザのツールバーに表示されるTab-Zen拡張機能アイコンのクリック。
*   **動作:**
    1.  クリック時、現在のブラウザウィンドウで開いている全てのタブ（ピン留めされたタブを含む）のURLとタイトルを取得し、セッションデータとして一時的に保存します。
    2.  保存が完了したら、元のタブを全て閉じます。
    3.  新しいタブを開き、Tab-Zenのミニマル画面（例: `tabzen.html`）を表示します。
    4.  ミニマル画面には、一時停止中のセッションからタブを復元するためのボタンを表示します。
    5.  **セッションの保存と復元:**
        *   セッションデータは、`chrome.storage.local`（Chrome Extensions API）またはブラウザのローカルストレージを用いて保存します。
        *   ミニマル画面上の「全てのタブを復元」ボタンをクリックすると、保存された全てのタブを新しいタブで開き直します。
        *   復元後、Tab-Zenのミニマル画面は自動的に閉じられます。
        *   保存されたセッションは、復元されるまで保持されます（現時点では単一セッション対応を想定）。
*   **UI/UX:**
    *   **拡張機能アイコン:** ブラウザのツールバーに表示されるTab-Zenロゴアイコン。
    *   **ミニマル画面:**
        *   背景はシンプルな単色（例: `#F8F8F8`）。
        *   画面中央下部、または視認性の高い位置に「全てのタブを復元」ボタンを配置。
        *   ボタンのデザインはシンプルで、直感的に操作できるものとします。
*   **考慮事項:**
    *   複数のブラウザウィンドウが同時に開いている場合、クリックした拡張機能アイコンが属するウィンドウのタブのみを一時停止・復元対象とします。
    *   元のタブのアクティブ状態やスクロール位置などの詳細な復元は、初期バージョンでは含めません（URLとタイトル情報のみでの復元を想定）。

### 4.2. 機能2: フローティング「思考スクラッチパッド」

#### 4.2.1. 機能概要
Tab-Zenのミニマル画面の中央に表示される、極めてシンプルな一時的なメモ帳です。思いついたアイデア、タスク、キーワード、あるいはただの頭の中のつぶやきなどを自由に書き留めることができます。このメモは自動保存されず、プレッシャーなく思考を吐き出し、整理するための安全で一時的な空間を提供します。

#### 4.2.2. 詳細仕様
*   **UI/UX:**
    *   ミニマル画面の中央に、シンプルなプレーンテキスト入力が可能なテキストエリアをフローティング表示します。
    *   テキストエリアは、ある程度の初期サイズを持ち、必要に応じてユーザーがリサイズできるようにします。
    *   プレースホルダーテキストとして「ここに思考を自由に書き出してください...」などを表示し、使用方法を促します。
*   **動作:**
    *   入力されたテキストは、明示的な保存操作なしにブラウザのメモリ上（または`sessionStorage`）に一時的に保持されます。永続的な保存は行いません。
    *   テキストはプレーンテキストとしてのみ扱われ、マークダウンやリッチテキストなどの書式設定はサポートしません。
    *   **データの消去トリガー:**
        1.  Tab-Zenのミニマル画面が閉じられた時（タブを閉じる、または「全てのタブを復元」機能で他のタブを開いた時）。
        2.  ブラウザのウィンドウ自体が閉じられた時。
*   **考慮事項:**
    *   「どうせ消えるから」という気軽さを重視するため、データ消去時の確認メッセージは表示しません。
    *   テキストエリアの文字数制限は設けない方針ですが、非常に大量のテキスト入力があった場合のパフォーマンスについては考慮が必要です。

### 4.3. 機能3: フォーカスアシストタイマー

#### 4.3.1. 機能概要
ミニマル画面に組み込まれた、シンプルなポモドーロ・テクニック（25分作業、5分休憩など）を支援するタイマー機能です。一時停止したデジタル空間で、タイマーを使って集中作業を促したり、短い休憩を取ることを支援します。外部からの通知や情報に邪魔されずに、本当にやるべきことに意識を向け、限られた時間を有効活用するためのガイドとなります。

#### 4.3.2. 詳細仕様
*   **UI/UX:**
    *   ミニマル画面の分かりやすい位置にタイマー表示エリア、開始/一時停止/リセットボタンを配置します。
    *   タイマーの残り時間はデジタル表示（例: `25:00`）。
    *   現在のモード（例: 「作業中」「休憩中」）を示す短いテキスト表示を行います。
*   **動作:**
    *   **デフォルト設定:**
        *   作業時間: 25分
        *   休憩時間: 5分
        *   長休憩時間: 15分 (4サイクルごとに長休憩)
    *   「開始」ボタンでタイマーがカウントダウンを開始します。
    *   「一時停止」ボタンでタイマーを一時停止し、再開ボタンで続きからカウントダウンを再開します。
    *   「リセット」ボタンでタイマーを初期状態（25:00）に戻します。
    *   **タイマー終了時の通知:**
        *   タイマー終了時に、控えめな通知音を再生します。
        *   タイマー表示エリアの色を変更する、点滅させるなどの視覚的な通知を行います。
        *   タイマーは次のサイクル（休憩または作業）に自動的に切り替わります。
    *   ブラウザがバックグラウンドに移動したり、Tab-Zenのタブが非アクティブになってもタイマーは継続して動作します。
*   **考慮事項:**
    *   ユーザーによる作業時間や休憩時間のカスタム設定は、初期バージョンでは含めません。シンプルさを最優先します。
    *   通知音は、集中を妨げないよう、短く控えめなものを選定します。
    *   タイマーの精度はJavaScriptの`setInterval`や`setTimeout`で実装し、誤差を最小限に抑えるよう配慮します。

## 5. 技術スタック（想定）
*   **フロントエンド:** HTML, CSS, JavaScript (Vanilla JS または軽量フレームワーク)
*   **ブラウザ拡張機能API:** Chrome Extensions API (Manifest V3を推奨)
*   **データ永続化:**
    *   セッションデータ（一時停止中のタブ情報）: `chrome.storage.local`
    *   スクラッチパッドデータ: `sessionStorage` (一時的保持のため)

## 6. 開発フェーズ（提案）

### 6.1. MVP (Minimum Viable Product) フェーズ
本仕様書に記載されている3つの主要機能（ワンクリック「全タブ一時停止」機能、フローティング「思考スクラッチパッド」、フォーカスアシストタイマー）をシンプルなUIで実装し、最小限の価値提供を目指します。ユーザーがデジタルノイズから離脱し、集中できる環境を即座に提供できることを優先します。

### 6.2. 改善フェーズ
MVPのリリース後、ユーザーフィードバックに基づき、UI/UXの改善、各種設定オプションの追加（例: タイマー時間のカスタム設定、背景色の変更）、複数セッションの管理機能などを検討します。
